Intro

Installable via npm install --save ipfs-repo-migrations, it can also be used directly in the browser.

Download

The source is available for download from GitHub. Alternatively, you can install using npm:

$ npm install --save ipfs-repo-migrations

You can then require() ipfs-repo-migrations as normal:

const ipfsRepoMigrations = require('ipfs-repo-migrations')

In the Browser

Ipfs-repo-migrations should work in any ES2015 environment out of the box.

Usage:

<script type="text/javascript" src="index.js"></script>

The portable versions of ipfs-repo-migrations, including index.js and index.min.js, are included in the /dist folder. Ipfs-repo-migrations can also be found on unpkg.com under

  • https://unpkg.com/ipfs-repo-migrations/dist/index.min.js
  • https://unpkg.com/ipfs-repo-migrations/dist/index.js

migrate

src/index.js

Main function to execute forward migrations. It acquire lock on the provided path before doing any migrations.

Signature of the progress callback is: function(migrationObject: object, currentMigrationNumber: int, totalMigrationsCount: int)

migrate(path: string, toVersion: int, options: Object): Promise<void>
Parameters
path (string) Path to initialized (!) JS-IPFS repo
toVersion (int) Version to which the repo should be migrated.
options (Object) Options for migration
Name Description
options.ignoreLock boolean? (default false) Won't lock the repo for applying the migrations. Use with caution.
options.repoOptions object? Options that are passed to migrations, that can use them to correctly construct datastore. Options are same like for IPFSRepo.
options.onProgress function? Callback which will be called after each executed migration to report progress
options.isDryRun boolean? (default false) Allows to simulate the execution of the migrations without any effect.
options.migrations array? Array of migrations to migrate. If undefined, the bundled migrations are used. Mainly for testing purpose.
Returns
Promise<void>:

revert

src/index.js

Main function to execute backward migration (reversion). It acquire lock on the provided path before doing any migrations.

Signature of the progress callback is: function(migrationObject: object, currentMigrationNumber: int, totalMigrationsCount: int)

revert(path: string, toVersion: int, options: Object): Promise<void>
Parameters
path (string) Path to initialized (!) JS-IPFS repo
toVersion (int) Version to which the repo will be reverted.
options (Object) Options for the reversion
Name Description
options.ignoreLock boolean? (default false) Won't lock the repo for reverting the migrations. Use with caution.
options.repoOptions object? Options that are passed to migrations, that can use them to correctly construct datastore. Options are same like for IPFSRepo.
options.onProgress function? Callback which will be called after each reverted migration to report progress
options.isDryRun boolean? (default false) Allows to simulate the execution of the reversion without any effects. Make sense to utilize onProgress with this argument.
options.migrations array? Array of migrations to migrate. If undefined, the bundled migrations are used. Mainly for testing purpose.
Returns
Promise<void>:
